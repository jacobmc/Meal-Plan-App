## Deviations from the plan (all deliberate — don't "fix" them back)

- **Validation errors are 400, not 422** — codebase `ValidationError` convention; routes use `safeParse` + throw, never raw `.parse()`.
- **Test auth helper is `setMockClerkUser`** (plan says `mockClerkUser`).
- **`normalizeUnit`**: plan's reference code had a bug — lowercasing the `"T"` alias clobbered the `"t"`→tsp mapping. Fixed with `if (a === "T") continue;` in the lookup construction.
- **Zod v4 rejects UUIDs with bad version nibbles** — test fixtures use v4-shaped UUIDs like `11111111-1111-4111-8111-111111111111`.
- **`INGREDIENT_CATEGORIES`** already existed in `@/lib/validation/ingredient`; `src/lib/grocery/types.ts` re-exports it rather than redefining.
- **No `date-fns`** (not a dependency; plan assumed it). UI uses `Intl.DateTimeFormat(..., { timeZone: "UTC" })`, `parseISODate`/`formatISODate`/`weekStartFor` from `@/lib/schedule/week`, and local `addDaysUTC` helpers.
- **Workbox `urlPattern` regexes anchored with `^/` never match** (RegExpRoute tests against the full href). All runtime-caching entries in `next.config.ts` use functional matchers on `url.pathname` with a `sameOrigin` guard instead of the plan's regexes.
- **`workbox-background-sync` was NOT installed** — unnecessary: workbox-build's generateSW bundles the BackgroundSyncPlugin automatically when `options.backgroundSync` is set (confirmed in `node_modules/.pnpm/workbox-build@7.1.0/.../types.d.ts`; functional `urlPattern` + `method: "PATCH"` also confirmed supported — this resolves the check flagged in the previous handoff).
- **`OfflineIndicator` queue count is inert**: it listens for SW message `grocery-checkoff-queue-status`, but the generated SW never posts that message (would need a custom `swSrc`). The offline banner itself works via online/offline events. Acceptable for v1; note it in the PR.
- **`[id]/page.tsx` reuses `loadItemsWithJoin`** from `src/app/api/grocery/_items.ts` instead of duplicating the join select.
- **`GroceryListIndex`**: "Show archived" toggle renders even when the list is empty (plan hid it, making archived lists unreachable from an empty index).
- **`GroceryListView` header count** computes checked/total from live client state, not the server-frozen `itemCount`/`uncheckedCount`.
- Grocery index unchecked-count query is family-scoped (innerJoin `grocery_lists`), unlike the plan's unscoped version.
